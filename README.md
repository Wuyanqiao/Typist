# Typist

一个 Windows 桌面打字助手，自动将草稿内容逐字符发送到任意目标输入框。

## 功能特性

### 核心功能
- **逐字符输入**: 使用 Win32 SendInput API 将文本逐字符发送到目标窗口
- **目标选择**: 点击选择目标输入框位置，支持多显示器
- **全局热键**: `Ctrl+Alt+S` 开始 / `Ctrl+Alt+Q` 停止
- **速度控制**: 可配置每字符间隔、随机抖动（模拟真人打字）、按键时长

### 新增功能 (v2.0)
- **多标签草稿**: 多个草稿标签页，同时管理多段文本
- **文本模板**: 保存常用文本为模板，一键加载复用
- **速度预设**: 慢速/正常/快速/极速 一键切换
- **智能进度**: 实时显示已输入/总数、预计剩余时间
- **字符统计**: 实时显示字数、行数、预计完成时间
- **文件导入**: 支持导入 .txt/.md/.csv/.log 文件
- **剪贴板粘贴**: 一键从剪贴板导入文本
- **拖放支持**: 直接拖放文本文件到草稿区
- **自动保存**: 草稿内容和设置自动保存，下次启动恢复
- **系统托盘**: 最小化到系统托盘，右键菜单快速操作
- **输入目标记忆**: 自动记住上次选择的目标位置
- **目标验证**: 开始前自动验证目标位置是否有效
- **输入重试**: SendInput 失败时自动重试（最多3次）
- **字体大小调整**: 草稿区字体可放大/缩小
- **输入历史**: 记录最近的输入会话

## 运行

### 从源码运行

```powershell
dotnet run --project .\Typist\Typist.csproj
```

### 构建单文件版本

```powershell
dotnet publish .\Typist\Typist.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -o .\artifacts\Typist-win-x64
```

## 使用方法

1. 在草稿区输入要发送的文本（或点击「导入文件」/「从剪贴板粘贴」）
2. 如需发送到特定位置，点击「选择目标」并在倒计时内将鼠标移到目标输入框
3. 调整输入速度（使用预设或手动设置间隔、抖动、按键时长）
4. 设置启动倒计时
5. 点击「开始输入」或按 `Ctrl+Alt+S`
6. 输入过程中可按 `Ctrl+Alt+Q` 或点击「停止」

### 速度预设

| 预设 | 每字符间隔 | 说明 |
|------|-----------|------|
| 慢速 | 100ms | 模拟缓慢打字 |
| 正常 | 60ms | 默认速度 |
| 快速 | 30ms | 较快输入 |
| 极速 | 0ms | 无延迟连续输入 |

### 随机抖动

「随机抖动」设置让每字符的实际延迟在基础间隔上下浮动，模拟真人打字节奏。设为 0 则使用固定间隔。

### 文本模板

- 点击「保存当前为模板」将当前草稿内容保存为可复用模板
- 模板按钮显示在模板区域，点击即可加载
- 右键点击模板按钮可删除模板

### 多标签管理

- 点击「+ 新标签」创建新的草稿标签
- 点击「- 删除」删除当前标签（至少保留一个）
- 每个标签的内容独立保存

## 设置

程序会在 `%APPDATA%/Typist/settings.json` 自动保存和恢复以下设置：

- 输入速度参数
- 窗口大小和位置
- 目标位置
- 草稿内容（多标签）
- 文本模板
- 各项选项开关

## 注意事项

- 使用 Windows `SendInput` API 模拟键盘输入
- 目标位置按屏幕坐标保存，窗口移动后需重新选择
- 如果目标程序以管理员权限运行，本程序也需要管理员权限
- 「换行使用 Enter」适合网页文本框和大部分多行输入框

## 技术栈

- C# / .NET 9
- Windows Forms
- Win32 API (SendInput, mouse_event, RegisterHotKey)
- System.Text.Json (设置持久化)
